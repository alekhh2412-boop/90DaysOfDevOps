# 90 Days of DevOps – Learning Plan

## Introduction
I am a fresher with basic knowledge of Linux and a strong interest in DevOps and Cloud Engineering.  
This 90-day plan is my commitment to build strong fundamentals and practical skills required for a DevOps role.

## Why I Am Learning DevOps & Cloud
I am learning DevOps to build a career in cloud infrastructure, automation, and modern software delivery.
I want to become job-ready by gaining hands-on experience with real DevOps tools and workflows.

## Where I Want to Reach (90-Day Goals)
1. Gain strong fundamentals in Linux, Git, and networking.
2. Build and understand CI/CD pipelines using GitHub Actions or Jenkins.
3. Deploy a containerized application using Docker and Kubernetes.

## Core DevOps Skills I Will Build
- Linux troubleshooting and command-line confidence
- CI/CD pipeline creation and debugging
- Containerization and Kubernetes basics

## Time Commitment
- Weekdays: 2 hours per day
- Weekends: 4–5 hours per day

## Consistency Strategy
I will study every day, track progress weekly, revise regularly, and share learning publicly to stay accountable.

## Commitment
I will focus on discipline and execution over perfection for the next 90 days.
